# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities.
2. Email **chad.hendren@gmail.com** with details and steps to reproduce.
3. Allow up to 72 hours for an initial response.

## Secret Handling

- Never commit API keys, tokens, passwords, or `.env` files.
- Use environment variables or your platform's secret manager.
- Rotate any credential that may have been exposed.

## Supported Versions

Security fixes are applied to the default branch (`main` or `master`).
