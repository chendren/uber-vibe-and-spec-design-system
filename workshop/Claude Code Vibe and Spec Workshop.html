Meet Claude
Code

An agent that lives in your terminal — it reads your code, makes the change, runs the command, with you in the loop the whole way.

Module 01 · 45 min

Module 01 · Mental model

It's a teammate, not autocomplete.

  • It reads the whole repoClaude opens files, follows imports, and builds context before it edits anything.
  • It acts in your toolsEdits files, runs tests, uses git — the same tools you'd reach for yourself.
  • You stay in the loopEvery change is a diff you review. Nothing ships without your eyes on it.
Coach's note

Stop asking "what prompt gets the answer." Start asking "what would I tell a sharp new teammate on day one."

Fast, conversational, exploratory. You discover the shape of the thing by building it out loud — and that's exactly the right move when the path is unknown.

Module 02 · 50 min

Module 02 · The vibe mode

How to vibe well.

  • Describe outcomes, not stepsSay what "done" looks like and let Claude choose the how.
  • Iterate in small loopsReact to what you see. Nudge, undo, try again — it's cheap to explore.
  • Keep momentumWhile you're exploring, perfect is the enemy of moving. Refine later.
Coach's note

Vibe is for unknowns — a new idea, an unfamiliar library, a throwaway spike. Don't over-spec what you don't yet understand.

03

Context &
tokens

Everything Claude knows in a session lives in one context window. Manage it on purpose and your results get sharper, faster, and cheaper.

Module 03 · 55 min

Module 03 · Foundations

The context window is the desk, not the library.

  • One finite spaceFiles, chat history, and tool output all share a single window.
  • What's loaded shapes the answerRelevant context in, sharp answers out. Junk in, junk out.
  • It fills upLong sessions drift as the desk gets cluttered. Clear it deliberately.
Spec note

Tokens are the unit of everything — input, output, and cost. Fewer, better tokens beats simply more tokens.

Uber Vibe & Spec12

Module 03 · Tactics

Spend tokens where they earn their keep.

  • Scope tightlyPoint Claude at the files that matter, not the whole repo.
  • Compact and resetUse /compact and fresh sessions to clear stale context between tasks.
  • Write durable facts downPut project rules in CLAUDE.md so you never re-explain them.
Coach's note

A short prompt with the right three files beats a giant prompt with the whole codebase pasted in.

04

Spec-driven
development

Write the plan, the constraints, and the tests first — then drive Claude precisely. This is how you hand off real autonomy and trust the result.

Module 04 · 60 min

Module 04 · Spec-driven

A good spec makes the agent unstoppable.

  • State the goalOne clear outcome, in plain language, before any code.
  • List the constraintsEdge cases, data shapes, and what must not change.
  • Write the tests firstLet failing tests define done — then let Claude make them pass.
Coach's note

The narrower the spec, the more autonomy you can safely hand the agent. Constrain first, then let go.

05

AI Skills for
Claude Code

Reusable folders of instructions and assets that teach Claude how you do a specific job — loaded only when the task calls for them.

Module 05 · 60 min

Module 05 · AI Skills

Teach Claude once. Reuse it everywhere.

  • A folder Claude can openA SKILL.md plus any scripts, templates, or references the job needs.
  • Loaded on demandClaude reaches for the skill only when the task calls for it — no context tax otherwise.
  • Shareable by defaultCheck skills into your repo so the whole team gets the same expert.
Coach's note

Anything you find yourself explaining to Claude more than twice is a candidate for a Skill.

Uber Vibe & Spec19

Module 05 · AI Skills

Name, description, and progressive detail.

  • Front matterA name and one-line description tell Claude exactly when to reach for it.
  • Plain-language instructionsThe body is how-to in the same words you'd tell a teammate.
  • Bundled assetsScripts, templates, and examples live beside it and load only if used.
Ties back to Module 03

Skills use progressive disclosure: a short description up front, the full detail only when invoked. That's token management built in.

Loops, tests &
your workflow

Keep the agent honest, then wire it all into a rhythm you can take back to your own codebase tomorrow.

Module 06 · 50 min

Module 06 · Verification

Trust the loop, verify the diff.

  • Tests are the contractGreen tests, not vibes, tell you a change is actually done.
  • Review every diffRead what changed. You own the code — the model doesn't.
  • Automate the guardrailsLint, types, and hooks catch what a tired reviewer misses.
Coach's note

Autonomy and verification rise together. The more you can check, the more you can safely hand off.

Module 06 · Putting it together

Vibe to find it. Spec to ship it.

  • Explore by feelVibe a spike until you understand the problem you're really solving.
  • Capture the specTurn what you learned into a plan, constraints, and tests.
  • Hand off and verifyLet Claude drive to green while you review the diffs.
The one thing

Switch modes on purpose. The skill isn't vibe or spec — it's knowing which one the moment needs.
