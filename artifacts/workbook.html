Module 03 · Spec-driven

Spec-driven development

In this module you'll turn a fuzzy feature request into a precise spec — then hand it to Claude and watch it build against your constraints and tests.

Learning objectives

  • Translate a vague request into a one-paragraph spec
  • Express constraints as tests and types the agent must honor
  • Drive Claude through implementation and review safely

Try it

# start from a blank spec
~ ❯ claude "draft a spec for password reset, then write failing tests"
✓ spec/reset.md · 5 cases · 8 tests (red)

Your turn

  • Write a 5-line spec for a feature in your own codebase.
  • Add two constraints the agent must not violate.
  • Generate failing tests, then let Claude make them pass.
